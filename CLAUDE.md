# token.app

AI model pricing tracker and comparison tool built on Cloudflare Workers with Hono. Data sourced from OpenRouter API. Deployed at https://token.app (custom domain) and https://token-app.measurable.workers.dev.

## Stack
- **Runtime**: Cloudflare Workers
- **Framework**: Hono
- **Storage**: Cloudflare KV (`TOKEN_APP_KV`)
- **Data**: OpenRouter `/api/v1/models` (JSON) + `/rankings` (RSC payload scraping)
- **Secrets**: `REFRESH_SECRET` via `wrangler secret put`

## Key files
- `src/index.ts` тАФ Hono routes (API + SSR page)
- `src/template.ts` тАФ Full HTML/CSS/JS template (SSR, single-file SPA)
- `src/fetchers.ts` тАФ OpenRouter data fetching, normalization, KV read/write
- `src/subscriptions.ts` тАФ Static subscription plan data
- `src/types.ts` тАФ All TypeScript interfaces
- `src/providers.ts` тАФ Provider metadata (colors, display names)

## Dev
- `npx wrangler dev --port 8799` тАФ local dev server
- `npx wrangler deploy` тАФ deploy to production
- Cron runs hourly (`0 * * * *`) to refresh model + rankings data
- Manual refresh: `POST /api/refresh` with `Authorization: Bearer <REFRESH_SECRET>`
- `wrangler.toml` is gitignored (contains KV namespace IDs)

## Gotchas
- **OpenRouter /rankings is JS-rendered now**: the SSR HTML no longer contains ranking data (turbopack Next.js app). `fetchRankingsFromOpenRouter()` uses the Cloudflare Browser Rendering binding via `@cloudflare/puppeteer`. Stable selector: `[data-testid="model-rankings-leaderboard-row"]` for models; Top Apps section has no testid (fallback to innerText regex). Requires `[browser]` binding in `wrangler.toml` and Workers Paid plan.
- **OpenRouter /rankings has NO period toggle**: the "This Week" / "Today" buttons in the new UI are model-search comboboxes, not period selectors. Models are fixed at 7-day rolling; apps are today-only. Don't try to click them to switch periods тАФ won't work and silently produces duplicate data. To support UI 24H/7D/30D toggle, we accumulate hourly snapshots in D1 and aggregate over windows ourselves.
- **D1 rankings history**: append-only `rankings_snapshots` table (`migrations/0001_*.sql`). Cron writes ~30 rows/hour. Read paths: `readLatestModels(env, 'week')` for any UI period (models don't change with toggle); `readLatestApps(env)` for day, `aggregateApps(env, days)` for week/month. Aggregation takes LAST snapshot per identifier per day then SUMs across days.
- **Week/month aggregations are gated behind real history**: `getRankings(env, 'week'|'month')` calls `countAppDaysInRange()` first and returns `topApps[period]=[]` plus `appsHistoryDays/appsHistoryRequired` when fewer than N distinct calendar days of snapshots exist. Without this gate, a 1-day SUM gets labelled "7D" and looks identical to 24H тАФ the "fake rankings" bug from 2026-05-28. Client (`emptyAppsMessage()` in `template.ts`) renders "X/Y days of history collected" so users see why the panel is empty. Real data lights up automatically as snapshots accumulate.
- **Empty-overwrite guard**: `refreshAllData` refuses to record a snapshot when scraping yields 0 models AND 0 apps. Stops silent cache poisoning if OpenRouter's UI changes again тАФ keeps last-good D1 data and surfaces `rankingsError` instead.
- **`[]` is truthy in JS**: `appsData.week || appsData.day` returns the empty `.week` array, not `.day`. The rankings UI fallback checks `byPeriod.length > 0` instead. Worth remembering for any future period-keyed shape.
- **Template literal escaping**: Model descriptions can contain backticks and `${` тАФ use `safeLiteral()` before embedding in template.ts.
- **`\'` is invalid in JS template literals**: backslash is silently ignored for single quotes. Use `this.hidden=true` instead of `this.style.display='none'` in inline handlers.
- **`/api/refresh` can't run the long category scrape**: HTTP `ctx.waitUntil()` is cancelled ~30s after the response (Cloudflare limit); only the *scheduled* (cron) handler has the 15-min budget for the ~2-3 min category scrape. `refreshAllData(env, { includeCategories })` gates it тАФ cron passes `true`, `/api/refresh` awaits the fast path (models+rankings+market-share, ~45-60s, under the ~100s edge timeout) and leaves categories to the cron. Backgrounding a long job after a 202 silently truncates тАФ don't.
- **Market-share legend: extract by CONTENT, not tag**: the `#market-share` rows are NOT reliably `<button>`-wrapped (OpenRouter changed the DOM; the spike's `<button>` assumption broke extraction тЖТ empty for 2 days). Walk up from each `a[href="/{author}"]` to the nearest ancestor whose text has a `%`, anchored on the link label. Reject non-author links (clean-slug regex + denylist) тАФ the section holds a `/models?fmt=cards` CTA next to the unnamed "Others" aggregate. Sanitize on read too (`isAuthorSlug` in `readMarketShare`). Also: `page.waitForFunction(string)` THROWS under `@cloudflare/puppeteer` тАФ poll with `page.evaluate(string)` instead.
- **Model token-usage chart source = `model-rankings-chart`, NOT `/models`** (verified 2026-06-28): the year-long per-MODEL chart is built from `model-rankings-chart` (52 weekly points; each week = its top-9 + an "Others" bucket). `/rankings/models?view=week` is the LEADERBOARD feed and only carries the last ~6 days тАФ it CANNOT drive a year chart. "Others" is genuinely ~40%/week and that MATCHES OpenRouter (confirmed against their own hover tooltip: Others is the big PINK base band, ~47%). `modelShareSeries` displays the UNION of every week's top-9 (canonicalized keys, for cross-week continuity) and keeps "Others" as `entities[0]` тЖТ the BOTTOM band. The old bug projected TODAY's top-9 backward тЖТ historical weeks rendered empty/grey (was mislabelled a "D1 depth" caveat).
- **"Top models by task" treemap = `task-spend` `spend` half, NOT `tokens`** (verified 2026-06-29): `/api/frontend/v1/rankings/task-spend` returns `{data:{spend,tokens}}`, each `{windowDays, macroCategories, tasks[]}`. OpenRouter's treemap tiles read "X% of all **spend**", so `taskSpendFromRaw` surfaces `data.spend` (classification 11.7% spend тЙа 9.76% tokens тАФ they differ). Tasks are tagged (`code:general_impl`) with NO label in the payload тАФ OpenRouter curates the human labels, mirrored in the `TASK_LABELS` map (`openrouter-json.ts`, fallback `prettyTag()`). It's a 30-day SNAPSHOT (no time-series) тЖТ store in KV like apps, NOT D1. Category palette sampled from OpenRouter: general `#f76b15`, agent `#6e56cf`, code `#30a46c`, data `#0090ff`. `deltaPp == share` in the payload (new taxonomy, prior window ~0) тАФ OpenRouter's own UI shows the same, so it's faithful, not a bug. Fetch is non-fatal in `refreshAllData` (catchтЖТempty) so a task-spend outage can't block the primary models/apps/share refresh. **UPDATE 2026-06-29: the treemap now exposes BOTH halves via a Spend/Tokens toggle** тАФ `taskSpendFromRaw` returns top-level=spend + `TaskSpend.tokens?`; tokens block reuses the SAME field names (`spendShare`/`spendShareOfTotal`), only values + per-task top-models differ (spend top=claude-4.7-opus, tokens top=hy3-preview).
- **`/api/*` edge cache lags the cron by HOURS** (2026-06-29): Cloudflare serves `/api/task-spend` & `/api/market-share` with `max-age` up to 14400 (zone override; routes set 3600) + `cf-cache-status: HIT`, so after the hourly cron updates KV the page keeps a stale snapshot тАФ this kept the new Spend/Tokens toggle HIDDEN for hours (visibility flips on `data.tokens` presence). Fix: client appends a 10-min time-bucket key (`fetch('/api/task-spend?b=' + Math.floor(Date.now()/600000))`) тАФ edge still caches within each window but data refreshes ~10min. A cache-BUSTED URL (`?_=ts`) proves KV is fresh while the plain URL serves stale. **Leaderboard + apps board are SSR-embedded (no client `/api/rankings` fetch on load).** **UPDATE 2026-06-30 (caching policy pass):** measured live тАФ the SSR page `/` is NOT edge-cached (no `cf-cache-status`; re-rendered per request from KV/D1), so the boards are already fresh to тЙд1 cron cycle (the earlier "governed by page HTML cache тЖТ lags hours" worry does NOT hold). The bust is now consolidated in `withBust(url)` (`template.ts`, **top-level** script scope so both `init()` and the DOMContentLoaded handler reach it; appends a 10-min bucket, `?b=` or `&b=` as needed) and applied to the 3 **live** fetches: `/api/rankings` (period/category toggle тАФ the previously-missed one, busted via `withBust(buildRankingsUrl(p))` + the SSR-fallback fetch), `/api/market-share`, `/api/task-spend`. **Slow** endpoints (`/api/models`, `/api/subscriptions`, `/api/rankings/categories`) stay un-busted (4h edge cache fine тАФ change rarely). Same policy block sits atop the API routes in `index.ts`. Spec: `docs/superpowers/specs/2026-06-30-caching-freshness-policy-design.md`.
